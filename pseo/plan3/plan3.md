💾 DATABASE QUERIES YOU'll Use Most
Get Page by Slug
sql
SELECT * FROM pages 
WHERE cluster = $1 AND slug = $2 
AND published_at IS NOT NULL;
Find Related Pages
sql
SELECT target_page_id FROM internal_links
WHERE source_page_id = $1 
AND link_type = 'related'
LIMIT 5;
Get All Pages for Sitemap
sql
SELECT cluster, slug, updated_at FROM pages
WHERE published_at IS NOT NULL
ORDER BY published_at DESC;
Check Cannibalization
sql
SELECT k1.keyword, COUNT(*)
FROM keywords k1
GROUP BY k1.keyword
HAVING COUNT(*) > 1;
Monitor Rankings (manual export from GSC)
sql
INSERT INTO keyword_rankings (keyword, position, impressions, clicks, ctr)
VALUES ($1, $2, $3, $4, $5)
ON CONFLICT (keyword) DO UPDATE
SET position = $2, updated_at = NOW();

⚠️ CRITICAL DON'Ts
❌ Don't create pages without checking for cannibalization first
❌ Don't publish pages with Core Web Vitals < 80
❌ Don't forget to add internal links (breaks hub-and-spoke model)
❌ Don't duplicate content across pages (always differentiate)
❌ Don't ignore Google Search Console data (it's your feedback loop)
❌ Don't rush to 1,000 pages (quality > quantity)
❌ Don't neglect Core Web Vitals (Google prioritizes this heavily)
❌ Don't write for keywords first (write for users first)
